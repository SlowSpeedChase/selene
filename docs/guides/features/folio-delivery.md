# Folio Delivery (iPad)

**What this does for you:** Read any Selene document on your iPad and mark it up with the Apple Pencil — scan one QR code from your Mac and it opens in a clean reader over your home WiFi.

## Using it

Folio is a small reader app that lives in its own folder on your Mac (`~/folio`). It serves your documents to the iPad over your local network — nothing leaves your house, no cloud account.

To open a document on the iPad:

1. On your Mac, go to the folio folder and run the command with the file you want to read:

   ```
   cd ~/folio
   npx ts-node scripts/send-ipad.ts reports/2026-05-25-selene-session.md
   ```

2. A **QR code prints right in the terminal**, along with the web address it points to. It looks roughly like this:

   ```
   Starting folio on port 3000...

   Scan to open on iPad:

   [QR code block]

   http://192.168.x.x:3000/view/2026-05-25-selene-session.md

   Annotate with Apple Pencil. Tap "Send feedback" when done.
   Press Ctrl+C to stop.
   ```

3. **Point your iPad camera at the QR code** and tap the banner. Safari opens the document, formatted for the iPad (larger type, an annotation canvas).

4. **Read and annotate** with the Apple Pencil. When you're done, tap **"Send feedback"** to save your notes and marks.

5. Back on the Mac, **press Ctrl+C** in the terminal to stop folio. (Leave it running as long as you want to keep reading.)

**Where your annotations go:** when you tap "Send feedback" on the iPad, your typed text and pencil marks are saved as a Markdown file in a `feedback/` folder next to the document you opened. The file is named with the date and time (for example `2026-05-25T14-30-00-note.md`) and records which document you marked up. iPad annotations are **saved to that feedback folder for you to review** — they are not automatically pushed back into the Selene database. Open the folio project's `feedback/` folder to see what you wrote.

> **A note on Kindle:** delivering documents to a Kindle is a separate path (`scripts/send-report.ts` in the folio project) and is not covered here. This guide is the iPad flow only. Selene also ingests *handwritten Kindle Scribe notebooks* via a different pipeline — see **`docs/guides/features/capturing-notes.md`**.

## How it works

This feature spans **two separate projects** on your Mac. Knowing which is which makes troubleshooting much easier.

### The folio project (`~/folio`) — the reader and the QR code

This is a standalone project, **not** part of the Selene repo.

- **The command:** `~/folio/scripts/send-ipad.ts`. It:
  1. Checks the file you named exists.
  2. Finds your Mac's local network (LAN) address from `os.networkInterfaces()`. It **prefers a home-WiFi address** (`192.168.x.x`, `10.x.x.x`, or `172.16–31.x.x`) over a Tailscale VPN address (`100.x.x.x`).
  3. Starts the folio web server as a background process (`npx ts-node src/server.ts`).
  4. Waits for the server to come up, then prints the QR code and the URL.
  5. Blocks until you press Ctrl+C, then stops the server.
- **The QR code** is generated by the `qrcode-terminal` library and drawn with text block characters right in your terminal:

  ```ts
  import qrcode from 'qrcode-terminal';
  // ...
  qrcode.generate(url, { small: true });
  ```

- **The LAN reader server:** `~/folio/src/server.ts`. The document URL hits its `GET /view/*` route. The server detects whether the request comes from a Kindle or an iPad (`src/detect.ts`) and serves the matching stylesheet (`ipad.css` for iPad). For iPad requests it also injects the Apple Pencil annotation canvas and `pencil.js`.
- **The feedback endpoint:** `POST /feedback` (also in `~/folio/src/server.ts`). When you tap "Send feedback," the iPad posts your text plus the pencil strokes (converted to an SVG path) here. For **iPad** requests the server saves the result to disk via `writeFeedback()` in `~/folio/src/feedback.ts`, landing in `<projectDir>/feedback/`.

### The Selene side (this repo)

Selene does **not** run the folio reader or the QR command. Selene's folio-related code handles the *Kindle Scribe* direction and is separate from the iPad reader flow:

- `src/workflows/eink-ingest.ts` — scans for handwritten Kindle Scribe PDFs (filenames starting with `folio__`) and OCRs them into Selene as notes with `capture_type: 'folio'`. This is the scanned-paper path, not the iPad reader.
- `src/workflows/folio-feedback.ts` — runs the *opposite* direction: it reads Selene notes captured as folio and writes feedback Markdown files back into the relevant folio project's `feedback/` folder.

There is no Selene workflow that automatically picks up the iPad's `feedback/*.md` files, so treat those as documents you review in the folio project.

## Configure & customize

The iPad reader's only real knob is the port, set in the folio project (not in Selene).

| Setting | How to change it | Default |
|---------|------------------|---------|
| Reader port | `FOLIO_PORT` env var (read by `~/folio/scripts/send-ipad.ts`; can also be set in `~/folio/.env`) | `3000` |

To run on a different port for one session:

```
cd ~/folio
FOLIO_PORT=3001 npx ts-node scripts/send-ipad.ts reports/2026-05-25-selene-session.md
```

The web address in the QR code is built automatically as `http://<your-LAN-IP>:<port>/view/<file>` — you don't configure the IP; the script detects it.

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `Port 3000 is already in use` and the script exits | Folio is already running, or another app holds the port. Stop the other instance, or pick a free port: `FOLIO_PORT=3001 npx ts-node scripts/send-ipad.ts <file>` |
| `Could not determine local IP. Are you connected to a network?` | Your Mac isn't on a network. Connect to WiFi and re-run. |
| `File not found: ...` | The path you passed doesn't exist. Use a path relative to where you ran the command, or a full path. |
| iPad won't load the page after scanning | The iPad must be on the **same WiFi network** as the Mac. Confirm the URL shows a home-WiFi address (starts with `192.168.`, `10.`, or `172.16`–`172.31`). |
| QR code points to a `100.x.x.x` address | That's a Tailscale VPN address. The script prefers a normal LAN address, but if only Tailscale is available the iPad must also be on the same Tailscale network to reach it. Connect the Mac to home WiFi and re-run to get a `192.168.x.x` URL. |
| Annotations don't appear in Selene | Expected — iPad annotations are saved as Markdown in `~/folio/<document folder>/feedback/`, not pushed into the Selene database. Open that folder to read them. |
| Terminal still running after you're done | Press **Ctrl+C** to stop folio; the script prints `folio stopped.` when it shuts down cleanly. |
