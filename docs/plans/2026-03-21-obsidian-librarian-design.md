# Obsidian Librarian — Export Redesign

**Date:** 2026-03-21
**Status:** Ready
**Scope:** 2-3 days

---

## Problem

The current Obsidian export is over-engineered and unusable. Each note becomes a massive markdown file with emoji status tables, ADHD badges, brain state analysis, sentiment scores, and processing metadata. The actual content gets buried. Every note is written to 4 separate folder paths, creating a sprawling vault that's hard to browse. The result: Chase doesn't open Obsidian.

## Goal

Redesign the export so Obsidian becomes a useful, actively maintained library. The LLM acts as a librarian — organizing notes, writing topic summaries, surfacing patterns, and maintaining a dashboard that makes the vault worth opening every day.

## Acceptance Criteria

- [ ] Notes export as one clean markdown file each (not four)
- [ ] Each note shows: title, content, essence, theme, concept links
- [ ] No emoji tables, ADHD badges, sentiment scores, or processing metadata
- [ ] Topic index pages generated by LLM for themes with 2+ notes
- [ ] Dashboard page with: recent notes, active topics, emerging patterns, quiet topics
- [ ] Full library curation runs every hour (export new notes + rebuild indexes/dashboard)
- [ ] LLM failures don't block note export (graceful degradation)
- [ ] Generated files (Dashboard, Topics) marked with frontmatter so system knows what it manages
- [ ] User-created files in vault are never deleted or overwritten
- [ ] Old vault content cleaned up (4-path structure replaced with flat Notes/)

## ADHD Check

- Reduces friction: one place to look (Dashboard), not a folder tree to navigate
- Makes things visible: quiet topics surface forgotten threads
- Externalizes memory: the librarian tracks patterns you'd otherwise lose
- Reduces cognitive load: clean notes, not data-dense status tables

## Note Format

One file per note in `Selene/Notes/`. Filename: `YYYY-MM-DD-slug.md`.

```markdown
---
title: "Garden layout ideas"
date: 2026-03-21
theme: home-projects
concepts:
  - garden
  - outdoor-space
  - planning
---

# Garden layout ideas

> Thinking about moving the raised beds to the south side where they'd get
> more afternoon sun. The current spot is shaded by the fence after 2pm.

---

*Essence: Considering relocating raised beds for better sun exposure —
practical problem-solving about making the garden work better.*

*Theme: [[home-projects]] | Concepts: [[garden]], [[outdoor-space]], [[planning]]*
```

Rules:
- Content in blockquote to visually distinguish from generated text
- Essence from `distill-essences.ts` output (not mechanical first-2-sentences)
- Theme and concepts as `[[wiki-links]]` for Obsidian backlink navigation
- Simple frontmatter for search/filtering
- Write-once: notes are never modified after export

## Topic Index Pages

Live in `Selene/Topics/`. LLM generates and maintains them.

Every hour:
1. Query all processed notes grouped by theme
2. For themes with 2+ notes, ask LLM to write/update a topic page
3. Only regenerate topics that have new notes since last run
4. Topics without new notes in 30+ days get a "quiet" marker

A topic page:

```markdown
---
type: topic
updated: 2026-03-21
note_count: 8
---

# Home Projects

You've been thinking about your living space — mostly practical
improvements. The garden keeps coming up, especially sun exposure
and layout. There's also a thread about the garage workshop setup.

## Recent

- [[2026-03-21-garden-layout-ideas]] — Relocating raised beds for better sun
- [[2026-03-19-workshop-bench]] — Workbench dimensions and tool storage

## Earlier

- [[2026-03-08-fence-repair]] — South fence leaning, need to fix before spring

## Connections

Related topics: [[planning]], [[outdoor-space]]
```

Topic page behaviors:
- Summary paragraph is LLM-written from note essences
- Notes listed with essence as one-line description
- Split into "Recent" (last 2 weeks) and "Earlier"
- "Connections" links to related topics (LLM determines)
- LLM can consolidate overlapping themes into one page

## Dashboard

One file: `Selene/Dashboard.md`. Rebuilt from scratch every hour.

```markdown
---
type: dashboard
updated: 2026-03-21 13:00
---

# Selene Library

*Last updated: March 21, 2026 at 1:00 PM — 94 notes in the library*

## What's New
```

## Technical Design

### Two-Phase Export (runs hourly)

**Phase 1: Export new notes (no LLM)**
- Query `processed_notes` where `exported_to_obsidian = 0`
- Join with `raw_notes` for content, `processed_notes` for essence/theme/concepts
- Write one clean markdown file per note to `Selene/Notes/`
- Mark as exported in database
- Fast, no LLM dependency

**Phase 2: Curate the library (LLM)**
- Query ALL processed notes: themes, concepts, essences, dates
- Group by theme, compute stats (count, recency, new since last run)
- Generate dashboard: one LLM call with recent notes, topic stats, pattern context
- Generate/update topic pages: one LLM call per changed topic
- Write `Selene/Dashboard.md` and `Selene/Topics/*.md`

### Vault Structure

```
Selene/
  Dashboard.md              # The homepage (generated)
  Notes/                    # One file per note, flat (write-once)
    2026-03-21-garden-layout.md
    2026-03-19-workshop-bench.md
  Topics/                   # LLM-generated index pages
    home-projects.md
    work-planning.md
    health.md
```

### Guardrails

- Never delete files — only create or overwrite files with `type: topic` or `type: dashboard` in frontmatter
- Notes in `Selene/Notes/` are write-once (exported, then never touched)
- If Ollama is down or LLM fails, Phase 1 still runs — notes get exported, curation skipped
- Log which topics were regenerated and why

### Performance

- Phase 1: instant (database query + file writes)
- Phase 2: 1 dashboard call + ~5-15 topic calls = 30-60 seconds with local Ollama
- Acceptable for hourly cadence

### Migration

- First run should clean up old vault structure (4-path copies) or leave them in place with a note
- Existing `exported_to_obsidian` flags stay — don't re-export already-exported notes
- Re-export all notes in the new clean format by resetting the flag (one-time migration)

## What's NOT Included

- Action item extraction (low confidence in quality — revisit later)
- Folder hierarchy management (flat notes + smart linking instead)
- File moving/reclassification (too complex, breaks Obsidian links)
- Sentiment/energy/ADHD badges (noise, not signal)
- Multiple copies of notes (one file per note, period)

## Risks

- **LLM summary quality:** Mitigated by using essences (already validated) as input, not raw content
- **Ollama availability:** Mitigated by Phase 1/Phase 2 separation — notes always export
- **Topic drift:** LLM might create too many or too few topics. Start simple, tune prompts based on real usage
- **Migration:** Resetting export flags means ~100+ notes re-export. Run once, verify, then let hourly take over
